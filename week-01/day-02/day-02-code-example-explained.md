# Day 2 代码例子讲解

## 这段代码是什么
这是一段 **教学版最小示例代码**。

它故意把问题简化成：
- 已知少量 sparse correspondences
- 把它们写成 offset
- 暂时假设整张图共享同一个位移
- 直接构造 dense offset map

它不是比赛方案，不是 baseline 复现，也不是高性能算法。

---

## 出处
- 类型：教学版最小代码示例
- 来源：为这套学习计划专门编写

---

## 为什么选它
选这段代码，不是因为它“强”，而是因为它对 Day 2 最有教学价值：

1. 它最直接地展示了 sparse → dense 的过渡
2. 它能让你看见 offset map 的最小形态
3. 它没有工程噪音，适合第一次建立对象感
4. 它能作为进入 SpaceNet9 baseline 之前的缓冲层

---

## 它帮助理解 SpaceNet9 的哪一部分
它帮助你理解的是：

> **为什么 SpaceNet9 的最终输出对象会是一张 2 通道位移图。**

也就是从：
- 少量 tie-points / sparse correspondence

走向：
- dense displacement field / offset map

虽然真实 SpaceNet9 更复杂，但输出对象的核心表达逻辑是一致的。

---

## 代码逐段看什么
### 1. `sparse_offsets`
这里先把少量已知对应关系写成 `(dx, dy)`。

你要看到的是：
- correspondence 可以写成位移
- 位移是后续构造整图场的基础

### 2. `shared_dx, shared_dy`
这里用一个很粗糙的教学假设：
- 全图共享同一个位移

真实任务通常不会这么简单，但这个假设非常适合 Day 2：
- 因为它让你先看到“如何从点走向整图”
- 而不是先被复杂模型吓住

### 3. `offset_map = np.zeros((2, H, W), dtype=np.float32)`
这是今天最重要的一行之一。

它表示：
- 这张输出图有 2 个通道
- 第 1 个通道放 dx
- 第 2 个通道放 dy

这和 SpaceNet9 README 中描述的 2-channel transformation map 在表达形式上是同类对象。

### 4. 对两个通道赋值
这里把整张图每个像素都填成统一位移。

你要理解的是：
- 稀疏点的位移信息
- 被扩展成了整张图的位移表达

这就是 sparse → dense 的最小雏形。

---

## 阅读时重点看什么
- sparse correspondence 是怎么变成 offset 的
- offset map 为什么要有 2 个通道
- `(2, H, W)` 这种组织方式表达了什么
- 这个 toy 输出对象与真实 SpaceNet9 输出对象哪里相似

---

## 暂时不用纠结什么
- 为什么这里没有插值
- 为什么这里没有学习模型
- 为什么这里没有局部变化
- 为什么这里没有真实的 optical / SAR 输入

这些都不是 Day 2 的重点。

---

## 和真实项目相比省略了什么
这段教学代码省略了很多真实复杂度：
- 跨模态差异
- 局部非一致位移
- 训练/推理流程
- patch inference
- 评测脚本
- 更复杂的几何模型

所以它不是“真实方法”，而是“对象教学工具”。

---

## 读完后你应该能说出什么
> 这段代码虽然极度简化，但它让我清楚看到：少量已知对应关系可以被表达成位移，并进一步组织成一张 2 通道的 dense offset map；这正是 SpaceNet9 输出对象的核心形式。
