# 学习资料选用规范

这份文档定义这套教程以后如何选择、整理、解释学习资料。

目标不是“堆很多链接”，而是建立一个：

> **能服务 SpaceNet9 入门，并且每份资料都有明确用途的学习资料库。**

---

## 1. 总原则
只要能帮助理解 SpaceNet9，下面这些都可以选入：

- SpaceNet9 官方资料
- baseline / 获奖方案代码
- 其他图像配准项目代码
- 光流 / correspondence / warp / dense field 相关经典实现
- PyTorch 官方教程或高质量工程示例
- 论文附带实现中的关键片段
- 教学版最小示例
- 教学版伪代码

**但所有资料都必须回答 3 个问题：**
1. 它来自哪里？
2. 为什么选它？
3. 它如何帮助理解 SpaceNet9？

如果回答不清，这份资料就不应该被放进主学习路径。

---

## 2. 可接受的资料类型
### A. 官方/原生资料
直接来自 SpaceNet9 本身：
- README
- baseline
- scoring script
- 获奖方案说明与代码

**用途：** 保证任务理解不跑偏，建立最直接的项目闭环。

### B. 桥梁资料
不是 SpaceNet9 原始代码，但能帮助跨过理解断层：
- image registration toy example
- optical flow / displacement field 小例子
- homography / affine / TPS 教程片段
- tie-points → dense field 的教学示例

**用途：** 把抽象概念压缩成可理解对象。

### C. 工程基础资料
帮助进入 PyTorch / 推理 /数据流程：
- Dataset / DataLoader 示例
- tensor shape 示例
- patch inference 示例
- inference / evaluation skeleton

**用途：** 为读取 baseline 和方案代码做准备。

### D. 教学重写资料
包括：
- 教学版最小代码
- 伪代码
- 精简后的流程图
- 代码摘录 + 逐段解释

**用途：** 降低真实工程代码第一次接触的门槛。

---

## 3. 必须标注的元信息
以后任何被纳入教程的代码片段、伪代码、外部链接，都必须尽量附上下面这些信息。

### 最低要求
- **出处**：来自哪里
- **选入原因**：为什么值得学
- **与 SpaceNet9 的关系**：它帮助理解哪一部分

### 推荐附加项
- **类型**：原始代码 / 摘录代码 / 教学版示例 / 伪代码
- **阅读重点**：这次重点看什么
- **暂时不用纠结什么**：避免初学者陷入细节
- **省略了什么复杂度**：说明它和真实项目的距离

---

## 4. 学习资料卡片模板
建议以后所有资料都尽量按这个模板呈现：

```markdown
## 资料标题
- 类型：
- 出处：
- 链接/路径：
- 为什么选它：
- 它帮助理解 SpaceNet9 的哪一部分：
- 阅读时重点看什么：
- 暂时不用纠结什么：
- 和真实项目相比省略了什么：
```

---

## 5. 关于伪代码
伪代码可以用，而且应该用。

因为很多时候真实代码难，不是思想难，而是：
- 工程噪音多
- 参数和依赖多
- 文件跳转多
- 一次性出现太多细节

所以伪代码是合格学习资料，但必须说明：
- 它是不是原论文/原项目自带的伪代码
- 如果不是，是否为教学版整理
- 为了教学删掉了哪些复杂部分

### 推荐标法
- 类型：教学版伪代码
- 来源：根据某项目/论文实现整理
- 选入原因：原实现工程复杂度较高，不适合作为第一次接触材料

---

## 6. 关于代码示例的选择原则
代码示例不是越长越好，而要满足：

1. **短**：最好能在几分钟内读完
2. **清楚**：输入输出明确
3. **可运行或可讲清楚**：必要时用伪代码替代
4. **有映射**：能说明它和 SpaceNet9 的联系

### 优先级建议
- 第一优先：最小可解释示例
- 第二优先：高质量官方/教程片段
- 第三优先：真实工程代码摘录

不要一开始就把复杂工程代码整段扔给初学者。

---

## 7. 这套教程今后的统一编写要求
以后每个 day / 每个专题在加入学习资料时，至少要做到：

- 给出 1 个核心资料入口
- 给出 1 个代码示例或伪代码示例
- 标注资料出处
- 说明选入原因
- 说明它和 SpaceNet9 的关系
- 说明今天读这份资料时应该重点看什么

---

## 8. 不应选入主线的资料
以下资料即使“有点相关”，也不建议放进主线：

- 来源不明、质量不稳定的博客抄录
- 没有解释、只有大段复杂代码的仓库
- 和 SpaceNet9 几乎没有关系、只是名字相近的材料
- 对初学者不友好且无法说明学习目的的论文实现

这类资料可以放到“扩展阅读”，不要放到主线任务里。

---

## 9. 一句话标准
如果一份资料不能回答：

> **它为什么值得现在学，以及它如何帮助我更接近 SpaceNet9 baseline/源码？**

那它就不该进入主学习路径。
